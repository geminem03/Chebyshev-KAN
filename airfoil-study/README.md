# KAN vs. MLP: Iso-Accuracy Benchmark Study

Welcome to this benchmark study evaluating the **parameter efficiency and performance of Kolmogorov-Arnold Networks (KANs)** against traditional **Multi-Layer Perceptrons (MLPs)**.

The goal of this repository is to demonstrate how different neural network architectures perform when tasked with reaching strict **iso-accuracy thresholds** (such as **~93.5–94%**) on complex scientific regression data.

---

# The NASA Airfoil Task and KAN Suitability

This study utilizes the **NASA Airfoil Self-Noise dataset**, which consists of complex, continuous aerodynamic and acoustic sensor readings.

The objective is to predict the **scaled sound pressure level (in decibels)** generated by various airfoil blade sections tested in an anechoic wind tunnel.

The **5-dimensional input vector** includes:

1. Frequency (Hz)
2. Angle of attack (degrees)
3. Chord length (meters)
4. Free-stream velocity (meters per second)
5. Suction side displacement thickness (meters)

The NASA Airfoil dataset is governed by underlying **low-dimensional fluid dynamics and acoustic physics**.

KANs are theoretically well suited for this type of data because:

1. **Mathematical Representation**  
   Their edge-based learnable functions excel at modeling the continuous nonlinear relationships inherent in aerodynamic systems.

2. **Parameter Efficiency**  
   By adapting the actual shape of activation functions, KANs can solve complex regression tasks using **far fewer parameters** than a standard MLP.

---

# The Three KAN Architectures

To ensure this benchmark is an objective comparison, it is important to understand how **MLPs and KANs distribute their trainable parameters**.

While their internal mathematics differ significantly, every model in this study solves the exact same task:

**Mapping a 5-dimensional input vector to a 1-dimensional output.**

The fairness baseline across all architectures is the **total number of trainable parameters optimized during backpropagation.**

---

## 1. Baseline: Multi-Layer Perceptron (MLP)

In a standard MLP, learnable parameters consist of:

- Weights on edges
- Biases at nodes

**Mechanism**

An MLP applies a linear transformation followed by a fixed nonlinear activation function such as ReLU or Tanh.

**Parameter Scaling**

To increase modeling capacity, an MLP must:

- Add more neurons (wider layers), or
- Add more layers (deeper networks)

Both approaches rapidly increase weight matrix size, leading to the **parameter explosion observed at high accuracy targets.**

---

## 2. B-Spline KAN (`models/MatrixKan.py`)

This model places **piecewise polynomial curves (B-splines)** on network edges.

Instead of learning a single weight per connection, it learns the **control points of spline functions** across a grid, allowing the network to adaptively shape nonlinear mappings.

This implementation uses a **matrix-based formulation** to accelerate spline evaluation.

---

## 3. Chebyshev KAN (`models/ChebyKANLayer.py`)

This architecture replaces splines with **Chebyshev polynomial expansions**, which are well known for their stability and approximation properties.

The network learns the **coefficients of Chebyshev polynomials** up to a fixed degree for each edge, providing smooth global function approximation.

Because Chebyshev polynomials are orthogonal and numerically stable, they are particularly attractive for **efficient hardware implementations.**

---

## 4. Fourier KAN (`models/fftKAN.py`)

This architecture models edge functions using a **Fourier series representation**.

By learning the amplitudes of sine and cosine harmonics, the network can efficiently capture **periodic or oscillatory physical relationships** present in telemetry and sensor data.

---

# Note on Fairness

KAN edges typically contain **multiple learnable parameters** (such as spline control points or polynomial coefficients), whereas MLP edges contain only a **single scalar weight**.

Therefore, comparing architectures by **layer width or node count** would bias the results toward KANs.

To maintain fairness, this benchmark compares models using:

**Total Trainable Parameter Count**

Each parameter represents one degree of freedom in the optimization space, regardless of whether it adjusts:

- an MLP weight,
- a spline coefficient, or
- a polynomial coefficient.

---

# Methodology: Iso-Accuracy Benchmark

To ensure a fair comparison between architectures, we conducted an **Iso-Accuracy Study**.

Instead of matching parameter counts or network depth, we defined **strict testing accuracy thresholds** and searched for the **smallest model that could reach them**.

This was performed using:

- **Optuna hyperparameter sweeps**
- Up to **200 training epochs**
- Pareto-front analysis to identify optimal parameter-accuracy tradeoffs

Because shallow MLPs struggled to reach high accuracy on this dataset, a larger search space was required to identify configurations capable of matching the KAN results.

---

# Iso-Accuracy Results (Target ≈ 93.5%)

Below are the **most parameter-efficient architectures** discovered for each model family.

At this accuracy level, KANs remain extremely compact, requiring **fewer than ~500 parameters**, while the MLP requires over **21,000 parameters**.

| Model | Architecture (FC Dims) | Minimum Params | Actual Accuracy | Parameter Reduction vs MLP |
| :--- | :--- | :--- | :--- | :--- |
| **Chebyshev KAN** | `[5, 16, 1]` | **384** | 93.61% | **55.1× smaller** |
| **Fourier KAN** | `[5, 12, 1]` | **445** | 93.53% | **47.5× smaller** |
| **B-Spline KAN** | `[5, 7, 1]` | **504** | 94.13% | **42.0× smaller** |
| **MLP** | `[5, 234, 78, 18, 1]` | **21,175** | 93.50% | Baseline |

---

# Conclusion

The results clearly demonstrate that **Kolmogorov-Arnold Networks provide massive parameter efficiency gains** over traditional MLPs when modeling complex physical datasets.

By replacing dense linear layers with **learnable edge functions**, KAN architectures—particularly the **Chebyshev KAN**—achieve the same predictive accuracy while using **40–55× fewer parameters**.

This makes KANs extremely attractive for:

- **embedded inference**
- **scientific computing**
- **hardware acceleration**

# Testcase Generation & Verification

Verification is driven by a **bit-exact Python fixed-point emulator**:

```
hdl_emulator_chebyshev.py
```

The emulator numerically mirrors the **SystemVerilog RTL** and produces:

- `.npy` reference vectors
- `.mif` FPGA memory initialization files

The verification testbench (`tb_layer.sv`) is fully parameterized and automatically configured using:

```
gen_tb_config.py
```

# Simulating the Airfoil Iso-Accuracy Model

The Chebyshev KAN model used for hardware verification corresponds to the **[5, 16, 1] architecture** discovered in the iso-accuracy benchmark.

Using the quantization pipeline, this trained model is converted to **fixed-point INT16 parameters** and exported as FPGA memory images.

The following steps verify the hardware implementation in **ModelSim**.

---

## Step 1: Prepare Hardware Test Vectors

First generate the quantized test vectors using the **quantization repository**.

This produces the required `.npy` and `.mif` files for each layer.

Expected directory structure:

```
experiments/quantization_results/chebyshev_testcase/layer_0_5x16
experiments/quantization_results/chebyshev_testcase/layer_1_16x1
```

---

## Step 2: Run Layer 0 (5 Inputs → 16 Outputs)

### Copy Memory Files

Copy the entire `layer_0_5x16` folder from

```
experiments/quantization_results/chebyshev_testcase/layer_0_5x16
```

into the root RTL directory. Copy `mem_init` within the `layer_0_5x16` folder to the root RTL directory.

### Generate Testbench Configuration

```bash
python gen_tb_config.py \
  --num_inputs 5 \
  --num_outputs 16 \
  --testcase_dir .
  --degree 3 \
  --requant_shift 5
```

### Simulate

```tcl
do run_layer.do
```

Expected output:

```
Accumulator Errors: 0 / 16
Requant Errors: 0 / 16
TEST PASSED
```

---

This verification confirms that the **hardware Chebyshev KAN layer produces bit-exact outputs** matching the Python fixed-point reference implementation.
